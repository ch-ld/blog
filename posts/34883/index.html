Hexo + Yelee 搭建个人博客

  • hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。本文主要讲解在只需要一点点计算机的基础下，一步步搭建出一个漂亮的博客。

总体步骤

  1. 下载Node.js软件
  2. 下载git软件
  3. 注册github账号，新建github仓库，开启github pages
  4. 下载hexo
  5. 建站
  6. 网站配置
  7. 下载主题
  8. 主题配置

准备工作


安装 Hexo 相当简单。然而在安装前，您必须检查电脑中是否已安装下列应用程序：

  1. Node.js

  2. Git

以上两个软件下载安装一直下一步即可，如遇问题自行解决（实在是解决不了的话，估计我也帮不上你）。
如果您的电脑中已经安装上述必备程序，那么恭喜您！我们可以继续下面步骤了。

安装Hexo


所有必备的应用程序安装完成后，即可使用 npm 安装 Hexo。

如果以上目录都有，那么基本上就完成了一半，接下来我们先了解经常用到的目录的含义。

source


资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。
_posts 通常我们的文章存储在此目录。

themes


主题 文件夹。Hexo 会根据主题来生成静态页面，可以下载各种各样的主题来美化我们的网站。

当然你的远程仓库必须得开启GitHub Pages，默认的主题可能比较丑，那我们可以在网上找到一个适合自己的主题。

主题


在网上找到一个随眼缘的主题，有哪些好看的主题？，选择自己喜欢的，下载下来即可。
我这里下载的是这款主题：Yelee，下载完成之后我把它放在\blog\themes目录下，将这个主题重命名为yelee。

主题配置


接下来就需要修改主题的配置文件了，因为一般默认的主题配置文件都是一些原作者的信息了，我们需要一一修改为自己的网站，所以还是任重道远，慢慢修改即可。
此配置文件一般是这个位置\blog\themes\yelee\_config.yml(当然前面要加上你们本地的目录)，这一步配置基本上比较简单这里就不一一说明。我的这个主题的配置可参考Yelee主题说明文档
